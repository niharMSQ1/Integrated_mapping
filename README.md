# Integrated mapping

Two separate folders:

- **`model_generator/`** – Code that connects to your PostgreSQL DB (using `.env` lines 13–17: `DB_NAME`, `DB_USER`, `DB_PASSWORD`, `DB_HOST`, `DB_PORT`), introspects tables, and **generates `app/models.py`** (SQLAlchemy 2.0 models).
- **`app/`** – FastAPI application. It uses **`app/models.py`** for the database models (either the hand-maintained version or the one generated by `model_generator`).

## Layout

```
POLICY ENGINE FastApi/
├── .env                    # Copy from .env.example; set DB_* (13–17)
├── .env.example
├── app/                    # FastAPI app (separate folder)
│   ├── __init__.py
│   ├── main.py
│   ├── models.py          # ← Generated by model_generator (or maintained by hand)
│   └── db/
│       ├── __init__.py
│       └── base.py        # Declarative Base used by models.py
├── model_generator/        # Model generating code (separate folder)
│   ├── ...
│   └── requirements.txt
├── scripts/
│   └── migrate.py         # Create all tables in DB (e.g. --db-name stakflo_dev_nihar)
└── README.md
```

## Generate `app/models.py` from the database

1. Copy `.env.example` to `.env` and keep at least:

   ```env
   DB_NAME=stakflo_dev
   DB_USER=stakflo
   DB_PASSWORD=stakflo@321
   DB_HOST=192.168.6.4
   DB_PORT=5432
   ```

2. Install generator deps and run (from project root):

   ```bash
   pip install -r model_generator/requirements.txt
   python -m model_generator --output app/models.py
   ```

   This connects to the DB, introspects the `public` schema, and overwrites **`app/models.py`** with SQLAlchemy 2.0 model classes.

3. Optional: custom `.env` path or output path:

   ```bash
   python -m model_generator --env .env --output app/models.py
   ```

## Migrate models to the database

Creates all tables from `app/models.py` in the target database (e.g. `stakflo_dev_nihar`).

1. Ensure `.env` has DB credentials (same as above). To use a different database, set `DB_NAME` in `.env` or pass `--db-name`.

2. From project root:

   ```bash
   pip install -r model_generator/requirements.txt
   python -m scripts.migrate
   ```

   This uses `DB_NAME` from `.env`. To create tables in **stakflo_dev_nihar** without changing `.env`:

   ```bash
   python -m scripts.migrate --db-name stakflo_dev_nihar
   ```

   Tables created: organizations, policy_templates, users, org_policies, policy_versions, policy_approvers, roles, user_role_organizations, employees, certificates, organization_certificates, clauses, organization_policy_control_mappings, organization_certificate_controls.

## Run the FastAPI app

```bash
pip install fastapi uvicorn sqlalchemy asyncpg
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

- `app/models.py` is the single models file used by the app (generated or hand-maintained).
- `app.db.base` defines the declarative `Base` that generated and hand-written models use.
